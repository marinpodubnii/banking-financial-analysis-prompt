## üîß Getting Started

### 1. Fork the Repository
```bash
# Click the 'Fork' button on GitHub
# Then clone your fork
git clone https://github.com/YOUR_USERNAME/banking-financial-analysis-prompt.git
cd banking-financial-analysis-prompt
```

### 2. Create a Branch
```bash
# Create a descriptive branch name
git checkout -b feature/add-esg-analysis
# or
git checkout -b fix/correct-roe-formula
# or
git checkout -b docs/improve-kpi-guide
```

### 3. Make Your Changes
- Follow the existing structure and style
- Test your changes if applicable
- Update documentation as needed

### 4. Commit Your Changes
```bash
git add .
git commit -m "Add ESG analysis section to prompt"
```

Use clear, descriptive commit messages:

### 5. Push and Create Pull Request
```bash
git push origin feature/add-esg-analysis
```

Then go to GitHub and create a Pull Request.

## üìã Contribution Guidelines

### Prompt Modifications

When modifying the main prompt:

1. **Maintain Structure**: Keep the 10-section framework
2. **Add Value**: Ensure additions provide real analytical value
3. **Be Clear**: Use clear, professional language
4. **Test**: Test the modified prompt with AI models
5. **Document**: Explain why the change improves the prompt

### Documentation Standards

- **Clear Language**: Write for both experts and beginners
- **Examples**: Include concrete examples
- **Formulas**: Use standard mathematical notation
- **Sources**: Cite sources for regulatory requirements
- **Bilingual**: Provide both Romanian and English when possible

### Code of Conduct

- Be respectful and professional
- Provide constructive feedback
- Focus on the content, not the person
- Welcome newcomers and help them contribute

## üìù Style Guide

### Markdown Formatting
- Use `#` for main sections
- Use `##` for subsections
- Use `###` for sub-subsections
- Use **bold** for emphasis
- Use *italic* for definitions
- Use `code` for formulas and technical terms

### Financial Terms
- Use industry-standard abbreviations (ROA, ROE, NPL, etc.)
- Define terms on first use
- Be consistent with terminology

### Examples Format
```markdown
**Example:** 
If NPL ratio = 5.2% and Total Loans = 1,000 mil. MDL
Then NPL = 52 mil. MDL
```
